## 補足

MVP-1 により、

- 商品注文
- 一般注文
- 買掛注文
- 基本的な発注履歴管理

を開始できます。

MVP-2 により、

- 現場別運用
- 承認
- 価格改定管理

を追加し、FAX・メール中心運用の削減を開始します。

MVP-3 により、

- 現場別原価管理
- 月次集計
- ダッシュボード分析

を追加し、工事原価の把握を可能にします。

フェーズ2では、

- 発注先
- 外部基幹システム
- BI分析
- 在庫管理

との連携を行い、サプライチェーン全体へ拡張します。

# 開発費用軽減要素

| 要素                     | 内容                                          |
| ------------------------ | --------------------------------------------- |
| Laravel利用              | 認証、DB、権限、CRUD基盤を高速構築可能        |
| OSS活用                  | 管理画面、CSV、認証、通知等を既存資産利用可能 |
| 管理画面テンプレート利用 | UI制作工数を削減可能                          |
| 既存EC構造活用           | 商品・カート・注文処理を再利用可能            |
| AI活用開発               | CRUD・管理画面・帳票の実装効率向上            |
| 段階導入                 | MVP優先により初期費用を抑制可能               |
| CSV運用併用              | 初期段階で複雑連携を回避可能                  |
| クラウド利用             | インフラ構築費を抑制可能                      |
| 既存業務フロー踏襲       | 現場教育コストを抑制可能                      |
