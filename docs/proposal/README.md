## 最新コンセプト

現場から本社・管理側へFAXと電話で行っている発注を、まずはオンライン化します。

主目的:

- FAX/電話による発注依頼をWebフォームに置き換える
- 本社・管理側が発注内容をメールとPDFで受け取れるようにする
- 本格的な注文管理システムを導入する前に、現場運用を軽くオンライン化する

## MVP定義

MVP本体に含める内容:

- 現場担当者がWebフォームで発注内容を入力する
- 入力内容を本社・管理側へメール送信する
- 入力内容からPDF発注書を生成する
- PDFをメール添付、または管理側が確認可能な形で保存する

MVPでは、DBによる本格的な注文管理は必須にしません。まずはFAX/電話の置き換えを主目的とします。

## MVP+1

MVP+1は、FAX運用に近い受け取り体験を維持するための現場運用補助案です。MVP必須要件ではありません。

実装してもよい追加の一手:

- 生成したPDFを受注用PCに自動保存する
- 受注用PC側でPDFを自動プリントアウトする

構成案:

```text
Webフォーム
  -> PDF生成
  -> メール送信または保存
  -> 受注用PCがPDFを取得/同期
  -> 受注用PC側で印刷
```

サーバーから直接プリントする前提にはしません。まずは「PDF生成 -> 受注用PCが取得/同期 -> 印刷」の構成案として扱います。

未確定事項:
