# Risks

作成日: 2026-06-18

## セキュリティ上の注意点

現リポジトリは実装アプリ本体ではなく資料中心のため、アプリケーションコード由来の脆弱性評価は未実施です。

現時点で確認できる注意点:

- `docs/proposal/` 配下には、業務フロー、権限、DB概念、インフラ構成、見積・WBSなど、社外公開に適さない可能性がある情報が含まれます。
- `docs/proposal/_proposal__exports/` 配下にはPDF/HTMLのエクスポート成果物が含まれ、Markdown原本と同等またはそれ以上に共有されやすいため、公開範囲の確認が必要です。
- `.vscode/settings.json` に個人環境依存と見られる絶対パスが含まれます。秘密情報ではありませんが、環境情報の露出・再現性低下につながります。
- `package-lock.json` は存在しますが、今回 `npm audit` は実行していません。パッケージ更新禁止・ネットワーク制限の前提があるため、依存脆弱性は未確認です。

## 未管理ファイル

Git未管理ファイルとして以下を確認しました。

- `.DS_Store`
- `docs/.DS_Store`

これらはmacOSのメタデータファイルで、通常はリポジトリ管理・共有対象にしません。削除または `.gitignore` 追加の候補ですが、今回のAuditでは既存設定を変更していません。

## 公開してはいけない可能性のあるファイル

簡易探索で、以下のファイル名パターンは確認されませんでした。

- `.env`
- `.env.*`
- 秘密鍵・証明書系: `*secret*`, `*credential*`, `*key*`, `*.pem`, `*.p12`
- DBダンプ・ローカルDB系: `*.sql`, `*.sqlite`, `*.db`
- ログ・アーカイブ系: `*.log`, `*.zip`, `*.tar`, `*.gz`

ただし、ファイル名ベースの確認であり、すべてのファイル内容を秘密情報検査したわけではありません。

## 資料と実装のギャップ

既存資料には Laravel Application、MySQL、ファイルストレージ、メール通知、CSV出力、バックアップなどの構成案が記載されています。一方で、現リポジトリには該当する実装や設定が確認できません。

リスク:

- 開発着手前の資料リポジトリなのか、実装リポジトリが別に存在するのか判断が必要です。
- 資料上の構成を前提に見積・設計・運用判断を進める場合、実装リポジトリや本番/検証環境の所在確認が必要です。
- READMEが空のため、初見の参加者がリポジトリの目的、資料生成方法、成果物の扱いを判断しづらい状態です。

## ドキュメント管理上の注意

- Markdown原本とPDF/HTMLエクスポート成果物が同一リポジトリで管理されています。どちらを正とするか明記されていません。
- エクスポート成果物の再生成手順が未確認です。
- `package.json` に生成用 `scripts` がないため、資料生成の手順が属人化する可能性があります。
